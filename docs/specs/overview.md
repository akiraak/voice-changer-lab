# リアルタイムボイスチェンジャー 入門

> このページは「配信で使えるリアルタイムボイスチェンジャーには**どんな種類**があり、**それぞれの特徴は何か**、**有名/人気のものは何か**」を A4 一枚で俯瞰するための入門ページ。
>
> 詳しい分類軸（技術アプローチ / 提供形態 / モデル形態の 3 軸クロスマトリクス）は [voice-changer-types.md](./voice-changer-types.md)、ツール個別の深掘りは各 spec を参照。
>
> 実測値・断定的性能評価は書かない（[CLAUDE.md](../../CLAUDE.md) 方針）。「一般にこう言われる」「公式が標榜する」は区別する。

## 0. このページの読み方

ボイスチェンジャーの種類分けは色々な切り口があるが、**配信で使う**という観点だと「**自分の環境で動かせるか**」「**いくらかかるか**」が一番先に効く。そこで本ページは**提供形態**で 3 タイプに分けて整理する。

| タイプ | 一言で | こんな人向け |
| --- | --- | --- |
| **1. ローカル OSS** | 無料・要 GPU・カスタマイズ自由 | GPU を持っていて、自前モデルを学習・差し替えたい |
| **2. ローカル商用ソフト** | 有料・GPU 軽め・プリセットで即使える | セットアップを最小化し、用意されたキャラ声で配信したい |
| **3. クラウド SaaS** | GPU 不要・要ネット・従量/月額 | ローカル GPU が無い、ノート PC やスマホ運用主体 |

> 「クラウド SaaS でリアルタイム配信に使える」と公式機能で裏が取れたものは現時点で **Voice.ai 1 件のみ**（[cloud-saas-realtime.md §4.1](./cloud-saas-realtime.md#41-判定サマリ足切り判断)）。クラウドはまだ本命選択肢ではない、というのが本ラボの一次判断。

技術アプローチ（信号処理 / ニューラル変換）やモデル形態（話者依存 / Zero-shot / プリセット）の違いは [§3](#3-補足-技術アプローチとモデル形態の違い) で軽く触れる。

## 1. リアルタイムボイスチェンジャーの 3 タイプ（提供形態別）

### 1.1 ローカル OSS（無料・要 GPU）

**特徴**

- ソースコードが公開されており、自分のマシンで動かす。GPU を使う前提のものが多い
- モデルの差し替え・自前学習・他ツールとの連携の自由度が高い
- 一方で、セットアップ・依存関係・モデル配布元のライセンスを自分で面倒を見る必要がある
- 配信用途では「**実行ホスト** + **声変換モデル**」を組み合わせて使う構成が事実上の主流

**有名/人気の代表ツール**

- **[w-okada/voice-changer](./w-okada-voice-changer.md)（VCClient）** — リアルタイム声変換の**実行ホスト**。RVC / so-vits-svc / Beatrice / MMVC / DDSP-SVC など複数モデルを切り替えて動かせる。配信用途で OSS を使う場合の事実上の入口。Windows / Mac / Linux / Google Colab 対応
- **[RVC](./rvc.md)（Retrieval-based Voice Conversion）** — 声変換モデルとして最も広く流通している系統の一つ。学習データ側の特徴量を kNN（faiss）で参照して置換する設計。コミュニティ配布のチェックポイントが豊富
- **[so-vits-svc](./so-vits-svc.md)（SoftVC VITS Singing Voice Conversion）** — 公式は**歌声変換**を強く志向（公式 README が "TTS is not supported" と明記）。配信用途にも転用される。`svc-develop-team/so-vits-svc` は 2023-11-11 にアーカイブ済みで、以降は派生フォーク（so-vits-svc-fork 等）に開発が移っている
- **[Beatrice](./beatrice.md)** — 「**超低遅延・低負荷・低容量**」を標榜（公式サイト [prj-beatrice.com](https://prj-beatrice.com)）。CPU シングルスレッド動作・約 35 MB を公式が打ち出す。VST 配布あり。RVC / so-vits-svc とは違い**キャラクター入りエディションを公式自身が配布**（つくよみちゃん / 刻鳴時雨 / OLUNE）

**候補プールに名前が残っているもの:** MMVC（多話者代表）、DDSP-SVC（DSP × ニューラルの歌声変換系）。優先度を上げたら [TODO.md](../../TODO.md) で個別調査タスクへ昇格する。

> **ライセンス注意:** 「コードが OSS」と「配布される学習済みモデルが自由に使えるか」は**別問題**。コミュニティ配布のチェックポイント（個人・キャラ声の `.pth` / `.index`）は配布者の規約・学習素材の規約で個別に判断する必要がある（[rvc.md §5.4](./rvc.md) / [so-vits-svc.md §5.4](./so-vits-svc.md) / [beatrice.md §5.3](./beatrice.md)）。

### 1.2 ローカル商用ソフト（有料・GPU 軽め）

**特徴**

- 買い切り / サブスクで配布されるインストール型のデスクトップアプリ
- GUI・サポート・あらかじめ用意されたプリセット声が提供される
- 内部実装はブラックボックス（信号処理寄り / ニューラル寄りは製品次第）
- 自前モデルの取り込み可否は製品ごとに異なる
- 配信での商用利用可否・収益化可否は**製品ごとに規約確認が必要**

**有名/人気の代表ツール**

- **Voicemod** — プリセット型のキャラ声/エフェクト系で広く知られる商用デスクトップ製品。配信者向けの導線（OBS 等への接続ガイド）が整っている
- **Voidol**（クリムゾンテクノロジー） — 国内系の声質変換デスクトップ製品。リアルタイム声質変換を打ち出している
- **Voice.ai のデスクトップクライアント** — クラウドと連動するハイブリッド型（[§1.3](#13-クラウド-saasgpu-不要要ネット) でも触れる）

> 本ラボでは商用デスクトップ製品の個別深掘りはまだ未着手。優先度を上げる場合は [TODO.md](../../TODO.md) の「候補プール」から昇格する。

### 1.3 クラウド SaaS（GPU 不要・要ネット）

**特徴**

- 変換処理をクラウド側で行い、結果をストリームで返す
- ローカル GPU が不要な代わりに、ネットワーク経由の遅延・通信品質・利用料が論点
- 「配信ソフトに流せるリアルタイム入力ストリーム」として実用になるかは SaaS ごとの提供形態次第
- 多くの SaaS は**リアルタイム TTS（テキスト → 音声）**は揃っているが、**マイク入力 → 別声 → ストリーム出力**まで揃っているのは現時点では稀

**横断棚卸しの一次判断:** [cloud-saas-realtime.md §4.1](./cloud-saas-realtime.md#41-判定サマリ足切り判断)（最終確認日 2026-05-12）

| SaaS | 配信用途で使えるか | 一言 |
| --- | --- | --- |
| **Voice.ai** | **A**: 公式リアルタイム配信経路あり | デスクトップアプリが VB-Audio Virtual Cable 経由で OS に仮想入力デバイスとして露出。"Compatible with Streamlabs OBS" を公式明記 |
| **Resemble AI** | C: 部分的 | リアルタイム TTS（200ms TTFS 公称）は揃うが、Speech-to-Speech は同期 REST / 最大 5 分 WAV のファイル入力 |
| **ElevenLabs** | C: 部分的 | `speech-to-speech/stream` は応答 SSE のみで、入力はファイル単位。公式デスクトップ / OBS プラグイン無し |
| **CoeFont** | E: 未確認 | VC LP は "Real-time" を標榜するが、技術仕様 / ToS 本文が公式に裏取れず（再裏取りが [TODO.md](../../TODO.md) に残課題） |

**結論:** 配信向けに公式機能でそのまま乗るのは **Voice.ai 1 件のみ**というのが現時点の一次判断。

> 後続候補（Respeecher / Altered AI / Supertone / Voicemod クラウド連携 等）は [cloud-saas-realtime.md §7.3](./cloud-saas-realtime.md) に名前のみ残してある。

## 2. ざっくり比較表

| 項目 | ローカル OSS | ローカル商用ソフト | クラウド SaaS |
| --- | --- | --- | --- |
| 初期費用 | 無料（要 GPU） | 買い切り / サブスク | 無料枠 + 従量 / 月額 |
| GPU 必須か | おおむね必要 | 軽量設計の製品が多い | 不要 |
| ネット必須か | 不要 | おおむね不要 | 必須 |
| 自前モデル | できる | 製品次第 | 基本不可（クローニングは別議論） |
| プリセット声 | 自分で用意 | 豊富 | 製品によりプリセット型 |
| 配信ソフト連携 | 仮想オーディオデバイス経由で確立 | 公式ガイドあり | Voice.ai のみ公式経路 |
| ライセンス確認 | コード+モデル別々に必要 | 製品 EULA | サービス利用規約 |
| 代表 | RVC / so-vits-svc / Beatrice / w-okada | Voicemod / Voidol | Voice.ai |

> 配信ソフトへの実際の繋ぎ方（仮想オーディオデバイス経由 P1〜P4 の経路）は [virtual-audio-devices.md §5](./virtual-audio-devices.md#5-配信用途での典型構成) と [obs-studio.md §5](./obs-studio.md#5-配信用途での典型構成p1p4--obs-レシピ) に集約してある。

## 3. 補足: 技術アプローチとモデル形態の違い

提供形態とは別の切り口。深く知りたくなったら [voice-changer-types.md](./voice-changer-types.md) §1（技術アプローチ）/ §3（モデル形態）を参照。

### 技術アプローチ（中身がどう動くか）

- **信号処理ベース (DSP)** — ピッチシフト・フォルマント変換などで波形を直接いじる。CPU で軽く動くが「別人の声に化ける」用途には向かない。**製品の中の 1 段として登場**することが多い
- **ニューラル変換** — 入力波形 → 中間表現 → ターゲット話者の波形をニューラルネットで学習する系統。**現代の主流**。RVC（Retrieval ベース系）/ so-vits-svc・Beatrice（コンテンツ表現 + 話者条件付け系）/ Diff-SVC（拡散モデル系）などのサブカテゴリがある
- **ハイブリッド** — 信号処理での前処理（ノイズ抑制等）+ ニューラル変換 + 後段エフェクトを組み合わせる構成。「ボイスチェンジャー製品」と呼ばれるものはほぼハイブリッド

### モデル形態（ターゲットの声をどう決めるか）

- **話者依存型** — ターゲット話者ごとにモデルを学習する。1 つの声を高品質に追い込めるが、声を増やすたびに学習が必要。**OSS 系の VC は基本ここ**（RVC / so-vits-svc / Beatrice はこのタイプ）
- **多話者 / 話者非依存型** — 学習済みモデル 1 つで複数のターゲット話者を切り替えられる（MMVC など）
- **Zero-shot / Any-to-any** — リファレンス音声を数秒〜数十秒与えるだけでその声に変換する（OpenVoice / XTTS / ElevenLabs Voice Cloning 等）。**第三者の声を許諾なく真似るリスクが高い**ため利用規約・倫理面の判断が重い。配信向けに低遅延ストリームで動く代表例はまだ確立していない
- **プリセット固定型** — ベンダーが用意したキャラ声をそのまま使う。商用デスクトップ製品でよく見られる（Voicemod / Voidol 等）
