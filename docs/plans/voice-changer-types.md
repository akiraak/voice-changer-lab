# 調査プラン: ボイスチェンジャーの種類

## 目的・背景

リアルタイム声変換のツール選定を進める前提として、まず**どんな方式・どんな立ち位置のツールが存在するか**の見取り図を作る。
個別ツールの遅延・音質・GPU要件などの実測比較は後続タスクで行うので、ここでは「全体像と分類」までをスコープにする。

このドキュメントが揃うと、

- 後続の個別ツール調査（RVC / w-okada / Beatrice など）で「どの分類のどの位置にあるか」を一言で説明できる
- 配信用途で**そもそも候補に入らない方式**（オフライン専用など）を早い段階で除外できる

## スコープ

含める:

- リアルタイム声変換の**技術的アプローチ**による分類（信号処理ベース / ボコーダ系 / ニューラル声質変換 など）
- **提供形態**による分類（ローカルOSS / ローカル商用 / クラウドSaaS）
- **入力モデルの考え方**（話者非依存 vs ターゲット話者ごとに学習 / Zero-shot 系 など）
- 各分類における**代表例の名前出しのみ**（個別の詳細仕様は `docs/specs/` 配下の個別ファイルに後で書く）

含めない:

- 個別ツールの遅延・音質の実測値（後続タスク）
- 具体的なセットアップ手順（後続タスク）
- オフライン専用ツール（プロジェクトスコープ外）

## 対応方針

成果物は `docs/specs/voice-changer-types.md` の1ファイル。
内容は以下の構成で書く:

1. 全体像（Mermaid もしくは箇条書きのマップ）
2. 技術的アプローチ別の分類と特徴
3. 提供形態別の分類
4. 学習・モデル形態の軸（話者依存 / 非依存、Zero-shot 等）
5. 配信用途で見るべき主な評価軸（後続調査の観点を列挙）

記述ルール:

- 各ツールの**遅延◯◯ms / GPU要求 / 音質**などの実測値は書かない（CLAUDE.md ルール）
- 「一般に〇〇と言われている」と「自分達で確かめた」は明確に分ける。本ドキュメントは前者のみ
- ライセンス・利用規約はツールごとの個別ページで触れる前提で、本ドキュメントでは方針だけ言及
