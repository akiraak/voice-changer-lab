# 入門用 overview.md 作成プラン

## 目的・背景

既存の [voice-changer-types.md](../specs/voice-changer-types.md) は **3 軸クロスマトリクス**（技術アプローチ × 提供形態 × モデル形態）で詳細に整理されており、後続の個別 spec を吊るすハブとしては機能しているが、**「ざっと種類と人気ツールを知りたい」**というユースケースには情報量が多すぎる。

> ユーザー要望（2026-05-14）:
> - 配信で使えるリアルタイムボイスチェンジャーにはどのような種類があり、それぞれの特徴は何か
> - それぞれの種類で有名/人気のあるものは何か

これに応えるため、**A4 で読み切れる入門ページ**を新規に作成する。

## 対応方針

- 新規ファイル `docs/specs/overview.md` を作成する。**voice-changer-types.md は残す**（詳細分類のハブとしての役割は維持）
- **主軸は「提供形態」**（ローカル OSS / ローカル商用 / クラウド SaaS）。ユーザーが「自分の環境で何が使えるか」「いくらかかるか」が一番先に効く軸であるため
- 各カテゴリで以下を 1 セットにする:
  - **特徴**（どんな人向きか・GPU 要否・料金感・カスタマイズ性）を 3〜5 行
  - **代表ツール**（有名・人気のもの）を箇条書きで 1 行ずつ
  - 詳細を知りたい人向けに既存 spec へのリンク
- 補助情報として、技術アプローチ（信号処理 / ニューラル変換 / ハイブリッド）とモデル形態（話者依存 / Zero-shot / プリセット）の**ざっくり違い**を 1 セクションで触れる（深掘りはしない）
- 実測値・断定的性能評価は書かない（[CLAUDE.md](../../CLAUDE.md) 方針）。「一般にこう言われる」「公式が標榜する」を区別する
- 既存資料の再利用が中心で、**新規の Web 調査は行わない**

## ファイル構成（案）

```
overview.md
├── 0. このページの読み方（A4 一枚で全体像をつかむための入門）
├── 1. リアルタイムボイスチェンジャーの 3 タイプ（提供形態別）
│   ├── 1.1 ローカル OSS（無料・要 GPU）
│   ├── 1.2 ローカル商用ソフト（有料・GPU 軽め）
│   └── 1.3 クラウド SaaS（GPU 不要・要ネット）
├── 2. ざっくり比較表（GPU 要否 / 料金 / カスタマイズ性 / 配信ソフト連携 / 代表）
├── 3. 補足: 技術アプローチとモデル形態の違い（軽く）
└── 4. 次に読むもの（voice-changer-types / 個別 spec / cloud-saas-realtime）
```
